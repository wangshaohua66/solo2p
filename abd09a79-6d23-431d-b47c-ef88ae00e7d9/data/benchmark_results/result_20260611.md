## 抓取性能估算 (5000 条/30 分钟)

- 目标: 5000 条公告 / 1800 秒 ≈ 2.78 条/秒
- 单条入库耗时: ~0.8 ms
- **理论入库吞吐**: ~1250 条/秒（纯 DB），远高于目标
- 实际瓶颈: HTTP 请求延迟 + 反爬限速（1.5s-4s/请求），worker_count=60 时最大约 20-40 条/秒
